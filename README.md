 # Правила

Алфавит исходной строки - `{0, 1}`. По мере выполнения операций в строке будут появляться и другие `дополнительные` символы.
 
В качестве `дополнительных` запрещено использовать символ `\`. 

Символ `@` - считается служебным, используется для обозначения пустой строки. Пустая строка входит в любую строку и ее первым вхождением считается позиция непосредственно перед началом строки.

На вход запрещается подавать пустую строку. Поведение интерпретатора в таком случае неопределено. 

 Интерпретатор будет работать с тремя файлами: 
 
 `input.txt` - строка входных данных
 
 `rules.txt` - набор правил замены подстрок для Нормального Алгоритма Маркова
 
 `output.txt` - строка, полученная после выполнения алгоритма
 
 В файле `rules.txt` каждая инструкция подается c новой строки. Можно оставлять пустые строки. Инструкции бывают двух типов:
 
 * `A -> B` - замена первого вхождения `A` как подстроки на `B`.
 
 * `A ->. B` - замена первого вхождения `A` как подстроки на `B` и завершение программы.

# Запуск интерпретатора

```./run.sh <file_inp> <file_rules> ``` - Запуск интерпретатора на входной строке из файла `file_inp` с правилами из файла `file_rules`.

Получившаяся строка-результат будет записана в файле `src/output.txt`.

# Описание примеров

* `example1` - Инвертирование. `0110101 -> 1001010`
* `example2` - Сортировка. `0110101 -> 0001111`
* `example3` - Удвоение. `0110101 -> 01101010110101`
